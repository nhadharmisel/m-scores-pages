# m-scores-pages

一个用于归档和发布多个独立 HTML 页面的仓库。

## 项目简介

`m-scores-pages` 旨在存储和托管各类独立 HTML 页面，这些页面通常用于展示、测试或临时性内容发布。仓库本身不绑定特定域名或网站，仅作为页面文件的静态存放与访问入口。

## 目录说明

```
m-scores-pages/
├── pages/          # 存放所有独立 HTML 页面文件
├── assets/         # 页面引用的公共资源（CSS/JS/图片等）
└── README.md       # 本文件
```

- `pages/`：每个独立页面为一个子目录或单文件，命名清晰以区分内容。
- `assets/`：页面共享的样式、脚本或媒体资源，避免重复。

## 页面归档说明

- 所有页面均为纯前端 HTML，不依赖后端服务。
- 页面之间相互独立，无跨页面依赖关系。
- 归档时请保持文件结构简洁，避免深层嵌套。
- 每个页面应包含必要的元信息（如标题、编码声明），以便直接访问时正常渲染。

## 维护说明

- 新增页面：在 `pages/` 下创建新目录或文件，并将相关资源放入 `assets/`（如需）。
- 更新页面：直接修改对应文件，注意保留原有兼容性。
- 删除页面：移除对应文件或目录，同时清理不再使用的资源。
- 提交时请附带简洁的提交信息，说明变更内容。

## 其他

- 本仓库不存储任何敏感信息、商业内容或违反平台政策的内容。
- 页面仅供归档与展示用途，使用者应自行确保内容合规。

如需贡献或反馈，请通过 Issue 或 Pull Request 联系维护者。
